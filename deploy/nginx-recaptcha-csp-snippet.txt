# Фрагмент для nginx: если reCAPTCHA не грузится, часто виноват заголовок Content-Security-Policy.
#
# Сайт грузит api.js через бэкенд на том же пути, что и API: GET /api/recaptcha-api.js?render=...
# (см. projectBeton-server/server.js). Первый script может уложиться в script-src 'self'.
#
# После загрузки api.js Google подтягивает код с www.gstatic.com — без него капча не оживёт.
#
# В script-src добавьте как минимум:
#   'self' https://www.gstatic.com https://www.recaptcha.net https://www.google.com
#
# В frame-src (значок / виджет v3):
#   https://www.google.com https://recaptcha.google.com https://www.recaptcha.net
#
# Пример только добавок (остальная политика у вас своя):
#   script-src 'self' ... https://www.gstatic.com https://www.recaptcha.net https://www.google.com;
#   frame-src ... https://www.google.com https://recaptcha.google.com https://www.recaptcha.net;
